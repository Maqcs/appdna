ApplicationImportConfiguration Properties

The ApplicationImportConfiguration type exposes the following members.

Properties

 

NameDescription
AutoAnalyseAllModules Specify true to start the analysis automatically after the import is finished.
GeneratedMsiFolder If the import needs to generate an msi during the import process, such as when importing an SFT or App-V file, the intermediate msi files will be saved to this location. If this is null, then no files are saved.
IceValidationConfig An optional configuration for ICE validation. Can be null.
ModulesToAnalyse A list of modules to automatically analyse
OutputFolder The full path to the folder into which the output of the import process is to be placed.
SaveDetailedLogs Specify true to save intermediate files. Useful for troubleshooting.
SynchronizationContext A SynchronizationContext used to control the thread on which notifications are raised.
